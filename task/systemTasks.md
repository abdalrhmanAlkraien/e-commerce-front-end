## Notes & Conventions

### Task Status Definitions

- **PENDING**: Task not started, waiting in queue
- **IN_PROGRESS**: Currently being worked on (one at a time)
- **COMPLETED**: Finished, tested, working correctly ✨ (all tests passed)
- **BLOCKED**: Waiting on dependency or external factor
- **FAILED**: Completed but has bugs/issues, needs fixing

### Token Tracking

- **Input Tokens**: Tokens sent to Claude (context)
- **Output Tokens**: Tokens generated by Claude (response)
- **Total Tokens**: Input + Output
- **Cost**: (Input/1M × $3) + (Output/1M × $15)
- **✨ Separate tracking for implementation vs testing**

### Fix Tracking

- **Fix Count**: Number of times task was fixed after completion
- **Fix Tokens**: Additional tokens spent fixing issues
- **Fix Cost**: Additional cost incurred by fixes
- **✨ Re-testing included in fix tracking**

### Estimated vs Actual

- **Estimated**: Pre-calculated expectations
- **Actual**: Real values after completion
- **Variance**: Actual - Estimated (for learning)

**✨ Testing Conventions:**

- **Test Scenarios**: Number of test cases per task (typically 3-6)
- **Pass Rate**: Percentage of scenarios that passed
- **Test Duration**: Time spent running automated tests
- **Test Cost**: Tokens/cost specifically for testing
- **Cannot mark COMPLETED without passing tests**

## Commands Reference

**Core Commands:**
- `/execute-task` - Start next pending task (with testing)
- `/continue-tasks` - Execute multiple tasks (with testing)
- `/review-progress` - Show detailed progress report (with test metrics)
- `/fix-task` - Fix a completed task (with re-testing)
- `/log-tokens` - Manually log token usage
- `/review-token-usage` - Detailed cost analysis (including test costs)

**✨ Testing Commands:**
- `/test-task [X.Y]` - Run or re-run tests for specific task
- `/review-tests [X.Y]` - Review detailed test results
- `/fix-tests [X.Y]` - Fix test scenarios only
